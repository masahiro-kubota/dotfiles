## Development Philosophy


### Git Commit Guidelines

適切なタイミングでのコミットを心がける：

1. **コミットするタイミング**
   - 機能が完成し、すべてのテストが通過した時点
   - コード品質チェック（lint、型チェック、フォーマット）がすべて通過した時点
   - ビルドが成功することを確認した時点
   - ユーザーが明示的にコミットを要求した場合のみ

2. **コミット前の確認事項**
   - `npm run check` または同等のコマンドですべてのチェックが通過すること
   - 新規作成・変更したファイルが意図通りであること
   - コミットメッセージが変更内容を適切に表現していること

3. **コミットしないケース**
   - 作業途中の未完成なコード
   - テストが失敗している状態
   - Lintエラーや型エラーが残っている状態
   - ユーザーから明示的な指示がない場合

4. **重要な注意事項**
   - 自動的にコミットすることは避ける
   - ユーザーが「コミットして」「commitして」などの明示的な指示を出すまで待つ
   - 機能の動作確認が完了していない段階ではコミットしない
